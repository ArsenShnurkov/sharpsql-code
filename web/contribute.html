Developers

Check out the project from the subversion repository. Look in the code if you can help. Write some code and post it to one of the project members. You can also write NUnit tests that work with the MS SQL Server but fail with the current version. Because Sharp-SQL should be compatible we need to fix it.

Documentation

You can also download the binaries and test it and write documentation in the wiki.

Translation

We need also translators. The resources for translation can you find in the subversion repository in the i18n directory. The translation should start first with alpha version.

Database users

Test your existing database applications for MS SQL Server with Sharp-SQL and report bugs.

Blogger

Write in any blogs about Sharp-SQL and link to this site.

Graphic Artist

We need a logo for the Sharp-SQL database. You can save your ideas in the wiki.
